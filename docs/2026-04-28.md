### 2.2 `run_benchmark.py` 加了**优雅中断**

新增 `SignalController`（44–86 行）：SIGINT/SIGTERM 命中时**先弹交互确认**，确认中断会写一条 `signal_interrupted` JSONL 记录再退出（exit 130/143）。F2 SWE-bench 单 run 7 分钟，万一卡住可以干净停下而不是 `Ctrl+C` 丢数据——这是从 F1 转 F2 的工程纪律提升。

新增 `--tasks-file` 让一族对应多个任务表（本地 vs SWE-bench Lite），`--live-log` 让 oracle 输出实时打到终端。

### 3.3 SWE-bench smoke 跑了一次（且失败了）

```
swe_lite__astropy__astropy-12907  pass=False  lat=407.6s  in=59707  out=6087  api=31
```

**这不是简单的 agent 失败，而是 oracle 自己解析失败**：
- `model_patch_chars: 892` ✅（agent 真的产了一个 patch）
- `tests_unchanged: True` ✅（没改测试）
- `returncode: 0` ✅（swebench harness 跑完了）
- `resolved: None` ❌（oracle 找不到 resolved_ids 或 unresolved_ids）

`_read_swebench_resolution` 在 `eval_root` + `eval_root/evaluation_results/**` 搜 `*.json` / `*.jsonl`，按 mtime 倒序找 `instance_id` 对应的 `resolved`/`unresolved` 字段。`resolved=None` 表示 either：
1. SWE-bench 把结果写到了**别处**（典型路径是 `<cwd>/<run_id>/<model>/<instance_id>/report.json` 而不是 `eval_root/*.json`）
2. 或者输出文件结构与 `_find_resolution` 期望的 key 名不匹配

**建议立刻做**：跑一次 smoke 后保留 `eval_root` 不清，手动 `find benchmark/.cache/swebench_eval/ -name '*.json*' -mmin -60` 看实际写到哪了，再修 `_read_swebench_resolution` 的搜索路径。在没修好之前 SWE-bench 路径上的 Q 全是 false negative。
